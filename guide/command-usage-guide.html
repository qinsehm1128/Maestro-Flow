核心工作流 16 commands

项目初始化、规划、执行、验证、协调、milestones 与 overlays。全局入口 /maestro 是智能协调器。

/maestro
智能协调器。根据用户意图和项目状态自动选择最优命令链。
/maestro "<intent>" /maestro continue /maestro -y "<intent>" /maestro status
衔接 init→ roadmap→ plan→ execute
/maestro-init
初始化 .workflow/ 目录，创建 state.json、project.md、specs/。项目起点。
/maestro-init /maestro-init --from brainstorm:ANL-xxx
brainstorm→ init→ roadmap
/maestro-brainstorm
多角色头脑风暴，产出 guidance-specification.md。可选的初始化前置步骤。
/maestro-brainstorm "<topic>"
brainstorm→ init --from
/maestro-roadmap
交互式路线图生成。轻量方案，适合需求明确的小型项目。
/maestro-roadmap "<desc>" -y
init→ roadmap→ analyze
/maestro-spec-generate
完整规范链（7 阶段），产出 PRD + 架构文档 + roadmap。适合大型项目。
/maestro-spec-generate
init→ spec-generate→ plan
/maestro-impeccable --chain build
UI 设计原型生成。可选步骤，产出 design-ref/ 和 design tokens。
/maestro-impeccable --chain build
roadmap·→ ui-design→ plan
/maestro-analyze
分析阶段。消费 roadmap + project.md，产出 context.md 和 analysis.md。支持多次分析。
/maestro-analyze /maestro-analyze 1 /maestro-analyze "topic" /maestro-analyze --gaps ISS-xxx /maestro-analyze -q "topic"
roadmap→ analyze↻ analyze→ plan
/maestro-plan
规划阶段。消费 context.md，产出 plan.json + TASK-*.json。支持 revise 和碰撞检测。
/maestro-plan /maestro-plan 1 /maestro-plan --gaps /maestro-plan --dir scratch/xxx
analyze→ plan↻ plan(revise)→ execute
/maestro-execute
执行阶段。消费 plan.json，产出代码变更和 .summaries/。逐个 plan 执行，plan 内 wave 并行。
/maestro-execute /maestro-execute --dir scratch/xxx
plan→ execute→ verify
/maestro-verify
验证阶段。消费 .summaries/，产出 verification.json。发现 gaps 时回退到 analyze。
/maestro-verify
execute→ verify→ review 或 gaps→analyze
/maestro-collab
多 CLI 协作分析。自动或手动选择多个 AI 工具并行分析同一主题。
/maestro-collab "<topic>" /maestro-collab "<topic>" --tools gemini,qwen,claude /maestro-collab -y "<topic>"
collab→ plan --dir
/maestro-quick
最短路径快速任务。跳过可选 agent，产出存放在 scratch/ 不影响主干。
/maestro-quick "修复 bug" /maestro-quick --full "重构 API" /maestro-quick --discuss "方案讨论"
/maestro-milestone-audit
里程碑审计。基于 artifact registry 检查 phase 覆盖度和跨 artifact 集成验证。
/maestro-milestone-audit
test→ milestone-audit→ milestone-complete
/maestro-milestone-complete
归档 scratch 到 milestones/，推进到下一里程碑。需 audit 通过。
/maestro-milestone-complete
milestone-audit→ milestone-complete→ analyze (next M)
/maestro-overlay
Overlay 管理。创建和管理覆盖层配置，用于临时修改或扩展工作流行为。
/maestro-overlay
/maestro-fork
分支管理。创建工作流分支以并行探索不同方案。
/maestro-fork

质量 9 commands

代码审查、自动测试、UAT、调试、重构、复盘和同步。测试三轨并行：spec/gap/code 智能路由。

/quality-auto-test
统一自动测试。智能路由 spec/gap/code 三轨，CSV 并行写测试 + 诊断迭代。
/quality-auto-test {N} /quality-auto-test {N} --max-iter 5 /quality-auto-test {N} --layer L0-L3 /quality-auto-test {N} --re-run
verify→ auto-test→ review
/quality-review
分层代码审查。支持 quick/standard/deep 三级深度。自动为 critical/high 发现创建 Issue。
/quality-review {N} /quality-review {N} --level deep
auto-test→ review→ test
/quality-test
会话式 UAT 验收测试。支持 smoke 模式和 auto-fix 循环。
/quality-test {N} /quality-test {N} --smoke /quality-test {N} --auto-fix
review→ test→ milestone-audit
/quality-debug
假设驱动调试。科学方法（假设→实验→验证），支持并行假设验证。
/quality-debug /quality-debug --from-uat {N} /quality-debug --parallel
test(失败)→ debug→ plan --gaps
/quality-refactor
技术债务治理。反思驱动的重构迭代。
/quality-refactor /quality-refactor <scope>
/quality-sync
文档同步。代码变更后同步相关文档。
/quality-sync --since HEAD~5
/quality-retrospective
质量复盘。回顾质量指标和改进机会。
/quality-retrospective
/quality-integration-test
集成测试。验证模块间协作的正确性。
/quality-integration-test
/quality-business-test
业务测试。从业务规则视角验证功能正确性，失败时创建 Issue（带 REQ 追溯）。
/quality-business-test

主干 Phase 管线

完整的 Milestone 生命周期：分析 → 规划 → 执行 → 验证 → 审查 → 测试 → 审计 → 完成。支持多次 analyze/plan，逐个 execute。

analyze↻ analyze→ plan↻ plan(revise)→ execute→ verify→ review→ test→ milestone-audit
1
/maestro-analyze
全量分析当前里程碑，产出 ANL artifact
2
/maestro-plan
消费 analyze 产出，生成 plan.json + TASK 文件，含碰撞检测
3
/maestro-execute
逐个 plan 执行，plan 内 wave 并行，产出 EXC artifact
4
/maestro-verify
验证执行结果。有 gaps 时回退到 analyze
5
/quality-auto-test → /quality-review → /quality-test
质量三件套：自动测试 → 代码审查 → UAT
6
/maestro-milestone-audit → /maestro-milestone-complete
审计通过后归档，推进下一里程碑

五种 Scope 使用模式

每个管线命令支持 4 种 scope，灵活组合分析-规划-执行。

A
analyze → plan → execute → verify
一步到位：默认覆盖里程碑所有 phases
B
analyze 1 → plan 1 → execute 1 analyze 2 → plan 2 → execute 2
逐 Phase：每个 plan 独立执行
C
analyze → plan 1 → execute 1 → plan 2 → execute 2 analyze "hotfix" → plan --dir → execute --dir
混合模式：全量分析 + 逐 phase 执行 + ad-hoc
D
analyze 1 → analyze 2 → plan → execute
统一规划：多次分析后一次规划执行
E
analyze "auth" → plan --dir → execute --dir
纯 scratch：无需 init/roadmap
